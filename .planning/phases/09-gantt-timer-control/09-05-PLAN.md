<objective>
Build the core interactive primitive of the gantt — `GanttBar` — and its supporting pieces: the drag tooltip and a dedicated entry-delete confirm path. GanttBar handles edge-resize, body-move, running-bar rendering (tick-driven right edge, stop icon, no right handle), selection, double-click-to-editor, right-click context menu, and keyboard delete. Because the existing `ConfirmDialog` is hardwired to `useDeleteTimer` (PATTERNS "No Analog Found"), this plan adds a parallel `useConfirmDeleteEntryStore` + `ConfirmEntryDeleteDialog` backed by `useDeleteEntry`.

Purpose: SC-3 (drag reschedules + persists via setTimestamps + zoom-aware snap + Alt free-drag) and SC-4 (running bar live to now + stop icon, double-click editor, select+delete). This is the heaviest interaction work; isolating it from the canvas assembly (09-06) keeps both within context budget.

Output: GanttBar (+ CSS + test), GanttDragTooltip (+ CSS), useConfirmDeleteEntryStore, ConfirmEntryDeleteDialog (+ CSS + test).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pointer drag → setTimestamps/setStart/setEnd IPC | Drag-derived timestamps cross into the SQLite write path |
| keyboard/context-menu delete → deleteEntry IPC | Entry id crosses into a destructive DB operation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-11 | Tampering | Body-move producing end < start during a two-call commit | mitigate | Body-move uses the atomic `setTimestamps` IPC (plan 09-01), which validates `startTs < endTs` at the repo; never two separate calls (Pitfall 3) |
| T-09-12 | Tampering | Drag targeting a running entry's end edge → setEnd ValidationError | mitigate | Running-bar right handle omitted from the DOM (Pitfall 2); end can only be edited after the stop icon fires useStopTimer |
| T-09-13 | Tampering | Drag-derived epochs out of valid bounds reaching the DB | mitigate | All commits pass through plan-09-01 Zod schemas (`SetTimestampsArgsSchema`/setStart/setEnd) at the IPC boundary |
| T-09-14 | Repudiation/Integrity | Accidental entry deletion | mitigate | Delete routes through a blocking confirm dialog (ConfirmEntryDeleteDialog); no silent delete |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No packages installed — RESEARCH §"Package Legitimacy Audit" confirms zero new deps. |
</threat_model>
