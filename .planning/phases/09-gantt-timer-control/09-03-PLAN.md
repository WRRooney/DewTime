<objective>
Build the renderer data layer the gantt and tab features sit on: the `useGanttEntries` viewport query, the `useCreateEntry` mutation (double-click entry creation), the `useActiveTabStore` (tab state + SQLite persistence), and the fix that closes the gantt invalidation gap across EVERY entry-mutating path so the gantt always refreshes. Also extend the test mock-api so component tests can stub the new IPC surface.

Purpose: Wave 3 gantt components consume these hooks/store directly (no codebase exploration). The invalidation fix (RESEARCH Pitfall 1) is the single most important wiring detail in the phase — without it, drag-commits, creates, stops, and deletes appear to "snap back" because the gantt query key (`['timeEntries','gantt']`) is not covered by the existing `['timeEntries','byTimer']` namespace key. This plan is the single owner of that closure: it patches all four mutation hooks (`useSetEntryTimestamps`, `useCreateEntry`, `useStopTimer`, `useDeleteEntry`) so downstream gantt components inherit a consistent invalidation contract.

Output: Two new hooks, one new store (+ test), the patched `invalidateAfterTimestampEdit`, the patched `useStopTimer` and `useDeleteEntry` onSuccess handlers, and an extended mock-api.
</objective>
