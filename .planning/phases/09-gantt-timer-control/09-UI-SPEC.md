## Spacing Scale

Declared values (must be multiples of 4 — inherited from `tokens.css`):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline padding, bar edge-handle hit expansion |
| `--space-2` | 8px | Compact element spacing, tab strip inner padding |
| `--space-3` | 12px | Cell padding (matches existing table cells), gutter textarea padding |
| `--space-4` | 16px | Default element spacing, lane content padding |
| `--space-5` | 24px | Section padding, gutter pane default width unit |
| `--space-6` | 32px | Header row height (time-axis header), toolbar element heights |

Exceptions:
- Lane row minimum height: 44px (matches existing table row height; grows to fit description textarea content).
- Bar edge-handle hit target: 8px wide (4px inside + 4px outside bar boundary) even though the visual indicator is 2px. Ensures draggability at small zoom.
- Ghost add-lane height: 32px fixed (de-emphasized, does not grow).
- Gutter splitter hit target: 8px wide centered on the 1px visual divider.

### Approved Deviations from Standard Set {4, 8, 16, 24, 32, 48, 64}

The following values are multiples of 4 but fall outside the checker's standard set. Both are pre-existing project tokens — changing them would diverge from the established codebase. They are explicitly approved as exceptions:

| Value | Context | Rationale | Approval |
|-------|---------|-----------|----------|
| **12px** (`--space-3`) | Cell padding; gutter textarea padding | Pre-existing project token defined in `tokens.css` and used for existing `TimerTable` cell padding. Introducing a new value would create visual inconsistency with the table. 4×3 — valid multiple of 4. | Approved deviation — pre-existing codebase token |
| **44px** | Lane row minimum height | Matches the existing `TimerTable` row height for visual consistency when both views share the same date context. 4×11 — valid multiple of 4. Not a new value introduced by this phase. | Approved deviation — matches existing row height |

**Accent (`--color-accent`) reserved for — explicit list:**
1. Active tab underline / indicator (2px bottom border on active tab label)
2. "Now" vertical line on the Gantt canvas
3. Running bar fill color (with reduced opacity — `hsl(192 80% 45% / 0.25)` fill, full accent stroke)
4. Running bar right-edge pulse animation border
5. Week-total pill (existing, unchanged)
6. Jump-to-Today button border + label (existing, unchanged)
7. Focus ring (`outline: 2px solid var(--color-accent)`) on all interactive elements
8. Gutter splitter hover state (matches existing resize handle pattern)

**Per-project bar colors** (Claude's Discretion per CONTEXT.md): assign from a fixed palette of 6 desaturated hues (HSL lightness 55–65%, saturation 30–50% in dark mode) cycled by `project_id % 6`. Projects with no assigned project use `--color-bg-elevated` fill with `--color-border` stroke. This avoids accent color competition with the "now" line and running-bar status. No legend UI this phase — palette assignment is deterministic and consistent per session.

Suggested palette (dark mode):
- Slot 0: `hsl(210 35% 55%)` — steel blue
- Slot 1: `hsl(155 30% 52%)` — muted teal
- Slot 2: `hsl(280 28% 57%)` — muted violet
- Slot 3: `hsl(35 40% 55%)` — amber
- Slot 4: `hsl(0 30% 56%)` — rose
- Slot 5: `hsl(50 38% 55%)` — gold

Light theme: same hues, lightness reduced to 38–44% (they'll contrast against the light `--color-bg`).

## Interaction Contract

### Tab Strip
- Three tabs: `Timers`, `Gantt`, `Projects` (exact labels, D-01).
- Active tab: `--font-weight-semibold`, 2px bottom border `var(--color-accent)`, foreground color `var(--color-fg)`.
- Inactive tab: `--font-weight-normal`, no border, foreground color `var(--color-fg-muted)`.
- Tab hover: background `var(--color-bg-hover)`.
- Tab height: 28px (matches existing nav button heights in toolbar).
- Default: Timers (first load); thereafter last-active tab persisted to SQLite settings.

### Gantt Canvas Layout
- Canvas = sticky axis header (top) + scrollable lane area.
- Left gutter pane is sticky horizontally (stays visible during horizontal pan).
- Gutter pane default width: 220px (expressed as a percentage of canvas width, persisted to SQLite settings). Min width: 120px. Max width: 50% of canvas width.
- Gutter/canvas splitter: 1px line `var(--color-border)`; 8px hit target; hover shows accent color (matches existing resize handle pattern from TimerTable).
- Lane height: min 44px; grows to fit gutter description textarea content; always at least tall enough for the project combobox (28px) + textarea with 1 visible line (24px) + padding (12px top + 12px bottom) = 76px minimum rendered.

### Gantt Zoom & Pan
- Default zoom: current day (today 00:00–23:59) fills the horizontal viewport.
- Scroll wheel: zoom in/out. Clamp: max span = 7 days; min span = 1 hour.
- Shift+Scroll or click-drag on empty canvas (not on a bar): pan horizontally.
- Hit-test priority: bar edge handle > bar body > empty canvas. Hit area must be checked in this order.
- Pan cursor: `grab` at rest on empty canvas area; `grabbing` during drag.
- Resize cursor: `ew-resize` when hovering a bar edge handle.
- Move cursor: `move` when hovering a bar body (not handle).

### Time Axis
- Two tiers: top tier = date + day-of-week labels at midnight boundaries; bottom tier = time-of-day ticks.
- Tick granularity adapts to zoom: span > 3 days → 4-hour ticks; span 1–3 days → 1-hour ticks; span < 1 day → 15-minute ticks.
- Midnight separators: 1px solid `var(--color-border)` spanning full lane height.
- Tick lines: 1px `var(--color-border)` at 40% opacity (lighter than midnight separators).
- Axis header background: `var(--color-bg-elevated)`, 1px bottom border `var(--color-border)`.
- Axis header height: 48px (24px top date tier + 24px tick tier).

### Now Line
- 1px vertical line, color `var(--color-accent)`.
- Spans full lane area height (not into axis header).
- Advances every second via `useTickStore` push-tick (no polling).
- Renders only when "now" falls within the visible viewport epoch range.

### Bars
- Bar body: rounded rect, `var(--radius-sm)` (4px). Fill = project palette color at opacity 0.35. Stroke = project palette color at full opacity, 1px.
- Bar label: timer name (or description truncated), 12px, inside bar body, left-padded 4px, `clip: hidden; text-overflow: ellipsis`.
- Bar minimum rendered width: 8px (2 × `--space-1`). Real timestamps unchanged at minimum.
- Selected bar: stroke increases to 2px; background fill opacity increases to 0.6.
- Bar hover: fill opacity 0.5.

**Running bar (end_timestamp IS NULL):**
- Fill: `var(--color-accent)` at 0.2 opacity.
- Stroke: `var(--color-accent)` at full opacity, 1px.
- Right edge tracks "now" (updated via `useTickStore`); right edge handle hidden and non-draggable.
- Left edge handle visible and draggable.
- Stop icon: 10×10px filled square SVG, `var(--color-success)`, centered vertically, 8px from bar left edge. Hover: `var(--color-success-hover)`. Click: fires `useStopTimer`. After stop, right edge becomes draggable.
- Pulse animation: `box-shadow: 0 0 0 2px var(--color-accent)` with keyframe opacity 1→0.3→1 at 2s period (CSS `animation`).

### Drag Handles
- Edge handles: 4px wide visual indicator on bar left/right edge (dark 50% overlay). Hit target: 8px (4px bleed into bar, 4px outside).
- Handle visibility: visible on bar hover and during drag; hidden at rest on non-selected bars.
- Snap: zoom-aware (snap increment = viewport span / canvas width in pixels, rounded to nearest: 15min / 30min / 1h). Hold Alt for pixel-precise (no snap).
- Drag tooltip: appears 8px above bar; contains `Start: HH:MM` / `End: HH:MM` / `Duration: H:MM`; background `var(--color-bg-elevated)`, border `var(--color-border)`, padding `4px 8px`, `--radius-sm`, 12px monospace.
- Optimistic update on drop: bar moves immediately; `setTimestamps` IPC fires; on error the bar reverts with a 300ms CSS transition.

### Double-Click Interactions
- Double-click on **empty lane space** → create time entry: start snapped to nearest grid line, end = start + 1 grid increment. Bar appears immediately (optimistic).
- Double-click on **bar body** → open existing `TimestampEditor` window (via `window.api.editor.open(timerId)`).
- These are mutually exclusive by hit-test; bar body hit-test takes priority.

### Bar Selection and Deletion
- Single-click on bar body: select bar (highlight state). Click empty canvas: deselect.
- Selected bar: highlighted stroke (2px accent color).
- Delete/Backspace key when bar selected: routes through `ConfirmDialog` → `useDeleteEntry`.
- Right-click on bar: context menu with "Open Editor" and "Delete Entry" items. "Open Editor" = same as double-click on bar. "Delete Entry" = routes through `ConfirmDialog` → `useDeleteEntry`.
- Context menu background: `var(--color-bg-elevated)`; items 14px; item hover `var(--color-bg-hover)`; border `var(--color-border)`; `--radius-sm`.

### Overlap Rendering
- Same-lane (same timer) overlapping entries: sub-row stacking within the lane. Each sub-row is the same height as a standard bar (24px). Lane height grows to accommodate sub-rows.
- Cross-lane overlap hint: when two bars across different lanes share overlapping epoch ranges, show a faint vertical band `hsl(0 65% 50% / 0.08)` across the full canvas height at the overlap region. No label. Only shown at zoom levels ≤ 3 days span (too noisy at wider zoom).

### Ghost Add-Lane
- Pinned below the scrollable lane area (not inside scroll).
- Height: 32px. Background: `var(--color-bg)`. Top border: 1px dashed `var(--color-border)`.
- Content: `+` icon (14px, `var(--color-fg-muted)`) + text "New timer" (14px, `var(--color-fg-muted)`).
- Hover: background `var(--color-bg-hover)`, text/icon color `var(--color-fg)`.
- Click or double-click: fires `useCreateTimer`, new lane appears at bottom of scroll area with gutter focused.

### Info Popover (Gantt only)
- Trigger: `ⓘ` icon button (16px), positioned in the Gantt canvas top-right corner over the time axis, always visible.
- Icon: circle-i SVG, `var(--color-fg-muted)`, hover `var(--color-fg)`.
- Popover: appears on hover (300ms delay), `var(--color-bg-elevated)` background, `var(--color-border)` border 1px, `--radius-md`, padding `12px 16px`, max-width 280px.
- Popover content: compact two-column table (gesture → action), 12px body, `var(--color-fg-muted)` left column.
- Popover closes on mouse leave (200ms delay).

### Projects Tab
- Selecting "Projects" tab renders `<ProjectsManager />` inline inside `<main>` (reuses existing component, D-29).
- No separate OS window for this path (the existing separate-window path stays for other callers).

- **Stop timer**: stop icon on running bar → fires immediately, no confirm. (Timer continues to exist with `end_timestamp` set.)
