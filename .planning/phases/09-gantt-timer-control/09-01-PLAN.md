## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | timeEntries.createEntry / setTimestamps (arbitrary epochs) | mitigate | `CreateEntryArgsSchema`/`SetTimestampsArgsSchema` positive-int + `.refine(startTs < endTs)` at IPC boundary; repo `setTimestamps` re-checks `startTs >= endTs` |
| T-09-02 | Tampering | createEntry referencing a non-existent timerId | mitigate | SQLite FK `time_entries.timer_id REFERENCES timers(id)` throws on invalid FK; error surfaced via wrapped IPC error |
| T-09-03 | Tampering | createEntry attempting a second running entry (NULL end) | mitigate | `endTs` is non-nullable in `CreateEntryArgsSchema`; repo `createEntry` never writes NULL — preserves single-running-entry FSM invariant (D-26/D-27 allow visual overlap only) |
| T-09-04 | Denial of Service | listInRange with reversed/zero-span range | mitigate | `ListInRangeArgsSchema.refine(fromEpoch < toEpoch)` rejects before any DB query |
| T-09-05 | Tampering | settings.gutter_width_pct out of bounds / invalid tab | mitigate | settings `SetArgsSchema` branch `z.number().min(0).max(1)` and `z.enum(['timers','gantt','projects'])` |
| T-09-06 | Tampering | drag-driven setTimestamps corrupting a running entry's end | mitigate | repo `setTimestamps` throws `ValidationError` when `end_timestamp === null`; running bar's end edge is also non-draggable in UI (plan 09-05) |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No packages installed this phase — RESEARCH §"Package Legitimacy Audit": "No new packages are introduced." No supply-chain surface added. |
